#+TITLE: Webconfig
#+DESCRIPTION: CLI-утилита для редактирования конфигурационных файлов через Web UI

Webconfig — Python CLI-утилита, запускающая локальный веб-сервер с интерфейсом
для редактирования конфигурационного файла. Поддерживает форматы TOML, JSON, ENV
и опциональную валидацию по JSON Schema.

* Установка

#+begin_src bash
git clone <repo-url>
cd webconfig
python3 -m venv .venv
.venv/bin/pip install -e .
#+end_src

Python 3.11+.

* Использование

#+begin_src bash
# Минимальный вызов (порт по умолчанию 8080)
webconfig config.toml

# С указанием порта
webconfig config.json --port 9090

# С JSON Schema (enum → выпадайки, constraints, дефолты)
webconfig config.toml --schema schema.json

# Тёмная тема
webconfig config.env --preset dark

# Без открытия браузера
webconfig config.toml --no-browser
#+end_src

Сервер стартует на ~127.0.0.1~, автоматически открывает браузер.
Остановка: ~Ctrl+C~.

* Поддерживаемые форматы

| Формат | Расширение | Парсинг                     |
|--------+------------+-----------------------------|
| TOML   | =.toml=    | =tomllib= (Python 3.11 stdlib) |
| JSON   | =.json=    | =json= (stdlib)                |
| ENV    | =.env=     | Собственный парсер             |

* JSON Schema

При указании =--schema= редактор использует схему для:

- =enum= → выпадающие списки (=<select>=)
- =default= → значения по умолчанию для отсутствующих ключей
- =title= → человеческие названия полей
- =description= → тултипы (атрибут =title=)
- =required= → обязательные поля (звёздочка + валидация)
- =minimum= / =maximum= → ограничения чисел
- =minLength= / =maxLength= / =pattern= → ограничения строк
